# Self Low Risk Docs Change

Goal: let Spec2Flow complete one small docs-governance improvement in its own repository.

Requirement:
- add a short docs governance navigation hint to the active documentation map
- clarify where active docs rules live and where archived plan rules live
- keep the change limited to markdown documentation and directly related docs guidance
- do not change schemas, runtime behavior, or collaboration automation

Preferred change shape:
- update `README.md` and `docs/index.md`
- point readers to `docs/structure.md` and `docs/plans/index.md` for docs governance rules
- keep the wording concise and implementation-facing

Success criteria:
- the change stays docs-only
- the repo still passes its existing documentation checks
